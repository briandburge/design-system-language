# Design Language guidelines for Figma Make

Use sentence case for all interface copy.

Use concise, stable labels. Prefer verb-led buttons when action is not obvious. Do not use placeholder text as the only form label. Errors must explain recovery. Empty states should orient the user and provide one next action.

Avoid generic marketing language and formulaic AI writing patterns.

Refer to `rules/rules.yaml` for rule IDs and severity.

# Contributing

## Propose a change

Open an issue that includes:

- the rule or pattern affected
- the problem being solved
- preferred and discouraged examples
- likely exceptions
- whether the change is breaking

## Rule structure

Every rule must include:

- a stable ID
- category
- title
- specification
- rationale
- severity
- preferred examples
- discouraged examples
- exceptions where needed

## Versioning

- Patch: clarification, typo, or non-behavioral correction
- Minor: new rule, component, prompt, or backward-compatible capability
- Major: changed behavior that may alter existing audit results

## House rule

This repository should comply with its own language rules.
